# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

### Do Not

❌ Open a public GitHub issue  
❌ Post on social media  
❌ Share exploit details publicly

### Do

✅ Report privately via GitHub Security Advisory  
✅ Contact maintainers directly  
✅ Give reasonable time to respond before disclosure

## How to Report

### Option 1: GitHub Security Advisory (Recommended)

1. Go to **Settings** → **Security** → **Report a vulnerability**
2. Use the private form
3. Include details about the issue
4. Expected response: 24-48 hours

### Option 2: Private Email

Send to: hello.finksystems@gmail.com

**Subject:** Security Vulnerability Report - MCP Security Documentation

**Include:**
- Clear description of the vulnerability
- Affected file(s) and location
- Steps to reproduce (if applicable)
- Potential impact
- Suggested fix (optional)
- Your name (optional, if you want credit)

### Option 3: GitHub Security Tab

1. If your repository has it enabled
2. Use the private reporting form
3. Include same information as above

## What to Report

### In Scope

✅ Errors in security documentation  
✅ Incorrect code examples  
✅ Misleading security advice  
✅ Incomplete compliance information  
✅ Outdated security practices

### Out of Scope

❌ Vulnerabilities in external dependencies  
❌ General suggestions (use Issues instead)  
❌ Questions about usage (use Discussions)  
❌ Third-party tool issues

## Response Timeline

| Step | Timeline |
|------|----------|
| Report received | Within 24 hours |
| Confirmation | Within 48 hours |
| Fix developed | Within 7 days (for critical) |
| Release | Within 14 days |
| Public disclosure | With patch release |

If we're unable to meet these timelines, we'll notify you.

## Disclosure Policy

After we release a fix:

1. We'll credit you in the release notes (if you want)
2. You're welcome to publicly disclose
3. We'll acknowledge your contribution
4. Your name will be in security researchers section (optional)

## Security Best Practices

This project focuses on **documenting** security best practices. If you find an issue:

1. Check that it's actually a documentation error
2. Verify against official sources (OWASP, CWE, NIST)
3. Provide references to support your report

## Questions?

- Check existing issues first
- Review GitHub Discussions
- Contact via security report form
